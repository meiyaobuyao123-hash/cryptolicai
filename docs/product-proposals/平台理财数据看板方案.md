# 平台级理财产品数据看板 — 方案

## 一、需求分析

当前的"热门理财产品"是全市场混排。用户还需要一个维度：**按平台看**。

每个头部平台（Aave、Lido、Binance Earn 等）下面有哪些理财产品，这些产品的收益分布和资金分布是什么样的。

## 三、数据来源

### DeFi 平台

| 数据 | API | 说明 |
|------|-----|------|
| 平台 TVL | DefiLlama `/protocol/{name}` | 单个协议的 TVL + 历史 + 链分布 |
| 产品列表 | DefiLlama `/yields/pools?project={name}` | 该协议下所有收益池 |
| 收益分布 | 上述数据聚合计算 | 按 APY 区间分组统计 |
| 资金分布 | 上述数据聚合计算 | 按资产/链分组统计 |
| 历史 TVL | DefiLlama `/protocol/{name}` | 包含历史 TVL 数据 |

#### 关键 API 示例

```
# 获取 Aave V3 的所有收益池
GET https://yields.llama.fi/pools
→ 过滤 project == "aave-v3"
→ 返回每个池子的: symbol, chain, apy, tvlUsd

# 获取 Aave V3 的协议详情
GET https://api.llama.fi/protocol/aave-v3
→ 返回: tvl, chainTvls, tokensInUsd
```

### CeFi 平台

| 数据 | 来源 | 说明 |
|------|------|------|
| 交易量 | CoinGecko `/exchanges/{id}` | 24h 交易量 |
| 产品列表 | 各交易所 API 或爬虫 | 理财产品列表和费率 |
| 储备金 | CoinGecko `/exchanges/{id}` | 公开的储备资产 |

CeFi 的产品级数据（每个理财产品的TVL和APY）没有统一API，需要：
1. 各交易所官方 API（需要 API Key）
2. 第三方数据聚合（如 CoinInterestRate.com）
3. 自建爬虫

## 四、数据看板的指标体系

### 收益分布看板

| 维度 | 指标 | 计算方式 |
|------|------|---------|
| 整体 | 平均 APY | 所有池子 APY 的加权平均（按 TVL 加权） |
| 整体 | 中位 APY | 所有池子 APY 的中位数 |
| 分布 | APY 频率分布 | 按 0-2%, 2-5%, 5-10%, 10%+ 分桶计数 |
| 趋势 | 加权平均 APY 30天变化 | 衡量收益率走势 |

### 质押金额（TVL）分布看板

| 维度 | 指标 | 计算方式 |
|------|------|---------|
| 按资产 | 各资产 TVL 占比 | USDT/USDC/ETH/BTC 各占多少 |
| 按链 | 各链 TVL 占比 | Ethereum/Arbitrum/Polygon 各占多少 |
| 按规模 | 池子规模分布 | $100M+ / $10-100M / $1-10M / <$1M |
| 集中度 | Top 5 池子 TVL 占比 | 衡量资金集中程度 |

## 五、实施方案

### 阶段一：平台详情页（DeFi）

1. 在排行榜中点击任一 DeFi 平台 → 进入平台详情页
2. 调用 DefiLlama API 获取该协议下所有池子
3. 前端聚合计算收益分布和资金分布
4. 展示：核心指标 + 收益分布图 + 资金分布饼图 + 池子列表

### 阶段二：平台对比

1. 用户可选择 2-3 个平台进行对比
2. 同时调用多个协议的数据
3. 表格化对比展示

### 阶段三：CeFi 平台详情

1. 接入交易所 API 获取产品列表
2. 或通过 Deep Link 引导用户去交易所查看

## 六、数据更新策略

| 数据 | 更新频率 | 缓存 |
|------|---------|------|
| 协议 TVL | 5 分钟 | 服务端缓存 |
| 池子列表 | 2 分钟 | 服务端缓存 |
| 收益分布 | 前端实时聚合 | 随池子数据更新 |
| 资金分布 | 前端实时聚合 | 随池子数据更新 |
| CeFi 产品 | 1 小时 | 各交易所 API 限速不同 |

## 七、产品价值

这个数据看板解决的核心问题：

1. **让用户看懂一个平台** — 不只看 TVL 排名，还能看到资金怎么分布、收益水平如何
2. **辅助投资决策** — 对比不同平台的收益分布，找到性价比最高的产品
3. **风险评估** — 资金集中度高的平台/池子风险更大
4. **趋势判断** — 收益率下降的平台可能在衰退，上升的可能有新机会
